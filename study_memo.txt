validatesはconfigに設定することができ。外側の開始は
$config = array()
となり、このかっこの中に、例えば
  'controller/action' => array()
を書いていき、この中にarrayでルールを記述していく。

ここで書いたルールは、controlelr似て呼び出すことができ、条件分岐でvalidation通っているかを確認する際は、
()野中は省略可能。

また、基本的にバリデーションの条件分岐は
if ($this->form_validation->run('signup') == FALSE)
{
  $this->load->view('form/myform');
}
else
{
  $this->load->view('form/formsuccess');
}
といった形で行う。もしvalidation configであらかじめ設定してあったら、runの引数はなしでも大丈夫。
ただし、同じバリデーションをまとめたり、よりリファクタリングするために複数引数とる場合は、configの中にも対応する引数と
一致するarrayを設定しておく。

基本的にフロント部分を大きく書き直す際は、事前にプロジェクトリーダーに確認を行い、そのうえで、書き換えてよければ
リファクタリングしてもいいし、もしダメだったら、そのままの状態でどのように編集を行っていくかを
考える必要がある。
